  1. Les techniques de l'information et de la communication(TIC):
    Les TICs regroupent l'ensemble des outils (matériels et logiciels) qui permettent aux utilisateurs de :
    • Communiquer et Transmettre
    • Accéder aux sources d’informations
    • Stocker
    • Manipuler
    • Produire
  2. Catégories et objectifs:
    • Les TIC ont 3 catégories :
      1. La téléinformatique: est la science des techniques permettant l’échange d’informations numériques entre plusieurs systèmes informatiques
      2. Télécommunication: domaine où les systèmes communicants ne sont pas nécessairement informatiques : traitement du signal, transmission analogique, les centres d'appels, le système GPS...
      3. L’électronique:les composants,l'équipement grand public tels que les télévisions, lecteurs...etc dvd etc....
    • Objectifs recherchés:
      1. La connectivité: permettre la communication entre plusieurs ordinateurs
      2. La modularité:utiliser un ensemble restreint d’appareils généraux
      3. Une implantation simple: solution générale qui peut être installée facilement selon différentes configurations
      4. Une utilisation facile: disponibilité d’outils de communication libérant les utilisateurs de la connaissance de la structure du réseau
      5. La fiabilité: détection et correction des erreurs
  3. Composants de TIC:
    1. La partie matérielle
      a). Pc portable, la télévision, le téléphone portable, la tablette interactive, appareils photos numériques, téléphones portables…etc
      b). Station de travail : On appelle station de travail toute machine capable d’envoyer des données vers les réseaux PC, MAC, SUN Terminal X,serveurs…etc
    2. La partie logicielle
      a). Les systèmes d’exploitation, les logiciels d’application et d’utilisation des différents composants matériels.
      b). Les réseaux d’internet ou mobiles.

  5. Initiation aux techniques de communication:

      Les objectifs de communication correspondent aux réactions que l‘on attend du récepteur. Un objectif de communication doit se définir selon 3 dimensions :
      • Au niveau « cognitif » la communication a pour but La prise de conscience l’attention, la notoriété et l’information au sujet d’une marque, d’un produit d’une idée, ou d’une entreprise, à ce stade, la communication est avant tout informative.
      • Au stade « affectif » c’est de Modifier ou renforcer l’image de produit en agissant sur les attitudes ou les opinions, que ce soit pour un produit ou une marque.
      • Au stade « comportemental » le but est de modifier les comportements en 1provoquant l’achat, l’essai.

      • Les partenaires de communication:
        - L’émetteur : il transforme le message en signaux.
        - Le récepteur : il reçoit les signaux et reconstruit le message.
      • Les vecteurs:
        - Message : l’émetteur émet un message qu’il va tenter de coder le plus parfaitement possible.
        - Médias : c’est le moyen utilisé, pour transmettre le message.
      • Les fonctions:
        - Codage : c’est la traduction du message en un ensemble structuré de signifiants.
        - Décodage : c’est l’interprétation du récepteur.
        - Réponse : l’avis du client.
        - Feedback : c’est la réaction du récepteur face au message reçu. Cette réaction

      • Bruit: il peut venir perturber les signaux pendant la transmission.

1.Réseau (informatique) Network :

Un réseau informatique est un ensemble d'ordinateurs et de terminaux connectés entre eux pour échanger des informations numériques selon des règles bien définies. La mise en réseau permet de relier des ordinateurs afin qu'ils puissent partager des ressources. Un réseau repose sur l'interconnexion et la communication entre les nœuds, qui sont les extrémités d'une connexion, et les processus, qui sont les programmes en cours d'exécution. Les réseaux informatiques ont évolué pour intégrer la voix et la vidéo en plus des données informatiques.

2.Les caractéristiques de base d’un réseau sont :

Les réseaux informatiques sont définis par plusieurs caractéristiques clés : la topologie, qui décrit comment les équipements sont interconnectés physiquement et logiquement, le débit, qui mesure la quantité de données numériques transmises par seconde, la distance maximale ou portée du réseau, qui dépend de la technologie utilisée, et le nombre maximal de nœuds pouvant être interconnectés.

3.Quelques terminologies en réseau :

▪ Station de travail : désigne toute machine qui peut envoyer des données vers les réseaux, telles que les ordinateurs personnels (PC), les Macs ou les terminaux SUN.
▪ Nœud : désigne une station de travail, une imprimante, un serveur ou toute autre entité pouvant être adressée par un numéro unique.
▪ Serveur : désigne une entité qui est le dépositaire d'une fonction spécifique, telle que la base de données, le calcul, le fichier ou le courrier électronique.
▪ Paquet : désigne la plus petite unité d'information qui peut être envoyée sur le réseau. Un paquet contient généralement l'adresse de l'émetteur, l'adresse du destinataire et les données à transmettre.
▪ Réseaux homogènes : désignent des réseaux dans lesquels tous les ordinateurs sont de la même marque ou du même constructeur, comme le réseau AppleTalk pour les ordinateurs Apple.
▪ Réseaux hétérogènes : désignent des réseaux dans lesquels les ordinateurs connectés sont de marques et de constructeurs différents, comme le réseau Ethernet qui utilise un câble de type coaxial.
▪ Débit : mesure la quantité de données numériques (bits) transmises par seconde (bps) sur un réseau.

5.Classification des réseaux :

▪ Le premier critère est la portée ou l'envergure, qui distingue les réseaux locaux (LAN), métropolitains (MAN) et étendus (WAN). Les LAN sont des réseaux locaux d'entreprise qui ne dépassent généralement pas quelques centaines de mètres, avec des vitesses de transmission de 10 à 100 Mb/s. Les MAN regroupent plusieurs réseaux locaux dans une même zone, pouvant relier des points distants de 10 à 25 km, et sont généralement basés sur des câbles coaxiaux. Les WAN sont des réseaux étendus gérés par des opérateurs publics ou privés, qui assurent la transmission des données sur de longues distances à l'échelle d'un pays ou de la planète.
▪ Le deuxième critère de classification est l'ouverture ou l'utilisation, qui distingue les réseaux intranet (internes à une entité organisationnelle), extranet (ouverts vers l'extérieur) et internet (réseaux publics interconnectés à l'échelle de la planète).
▪ Le troisième critère est le support de connexion, avec des réseaux filaires (câble coaxial, câble en paires torsadées, fibre optique, lignes téléphoniques) et des réseaux sans fil (ondes radios, infrarouges, satellites, Bluetooth, etc.).
▪ Enfin, le quatrième critère est le type d'organisation, avec les réseaux postes à postes (Peer to Peer) où chaque ordinateur peut partager ses ressources, et les réseaux clients/serveurs où les stations sont dédiées à une ou plusieurs tâches spécialisées et gérées par un administrateur.

6.Le type de topologie :

Il mentionne quatre types de topologies : bus, étoile, anneau et maillée :
▪ Les réseaux en bus utilisent un câble coaxial pour connecter les ordinateurs en série, chaque ordinateur étant connecté au bus via des connecteurs BNC.
▪ Les réseaux en étoile, quant à eux, ont toutes les stations connectées à un commutateur, qui renvoie les données vers tous les autres ports réseaux ou uniquement au destinataire.
▪ Les réseaux en anneau fonctionnent en utilisant une boucle où les ordinateurs communiquent chacun à leur tour. Ils utilisent la méthode d'accès à "jeton" où une trame de 3 octets circule dans un seul sens et les données transitent de stations en stations en suivant l'anneau.
▪ Le texte mentionne également l'inconvénient des réseaux en bus, qui est la limitation du débit à 10 Mbits/s, ce qui les rend moins adaptés aujourd'hui.

Composants matériel d’un Réseau informatique :

  Les Cartes réseau :
Ce sont des composants matériels permettant à un ordinateur de se connecter à un réseau. leurs rôle est crucial dans la communication entre ordinateurs permettant la transmission et la réception de données via des protocoles de communication tels que TCP/IP.
 Switch : :
 un switch est un équipement qui permet à plusieurs appareils de communiquer dans un réseau local en utilisant les adresses MAC, en stockant les informations dans une table de correspondance pour envoyer les données uniquement à l'appareil destinataire, Améliorent la gestion du trafic et la sécurité.
   hub:
Un hub est un composant de réseau qui transmet les données à tous les appareils connectés, contrairement à un switch. Cela signifie que lorsque des données sont envoyées sur le réseau, tous les appareils connectés reçoivent ces données .

modele OSI :

Le modèle OSI est un modèle de référence pour les communications de réseau qui divise les communications en sept couches, chacune étant responsable de la communication de données entre les appareils de réseau. Les couches sont: la couche physique, la couche liaison de données, la couche réseau, la couche transport, la couche session, la couche présentation et la couche application. Chaque couche fournit des services specifique pour facilitè la communication entre les appareils .

modèle TCP/IP :

le modele TCP/IP est un modèle de référence pour les communications de réseau, qui divise les communications en quatre couches principales : la couche liaison de données, la couche réseau, la couche transport et la couche application. Ce modèle est utilisé pour la conception et la mise en œuvre des réseaux informatiques, tels que :les réseaux Internet ,les réseaux sans fil. Il est utilisé avec les protocoles de communication de réseau, tels que le protocole Internet (IP), le protocole de contrôle de transmission (TCP) .

L'adressage :

Les communications en réseau sont controlé par des règles telles que l'adressage, L'adressage IP est un numéro qui identifie chaque ordinateur dans le réseau, avec deux modes d'adressage : IPv4 et IPv6. L'IPv4 utilise 32 bits pour l'adressage et est représenté en décimale, tandis que l'IPv6 utilise 128 bits et est représenté en hexadécimal .

dressage IPV4 :

la composition d'une adresse IPV4, se divise en 2, l'adresse réseau et une autre pour l'adresse hôte. La valeur des bits pour la partie réseau permet d'identifier les hôtes appartenant au même réseau. Le nombre de bits pour la partie hôte détermine le nombre de machines qui peuvent être connectées au réseau. Il exsiste notamment différents types d'adresses IPV4, comme l'adresse réseau et l'adresse de diffusion .

  • Introduction

    •   Le web et Internet sont souvent confondus, mais en réalité, le web est une des applications d'Internet. Le World Wide Web (WWW) est un système hypertexte public sur Internet qui permet de consulter des pages accessibles sur des sites à l'aide d'un navigateur. Les pages web sont reliées entre elles par des hyperliens, qui créent l'image d'une toile d'araignée. Le principe du World Wide Web est d'accéder à des documents textuels situés sur des machines accessibles par Internet et reliés entre eux par un mécanisme de lien hypertexte. Le web sert aujourd'hui des ressources de différentes natures : texte, image, son, vidéo, contenu applicatif et interactif, tels que l'achat ou la discussion.

  • Aspects techniques du Web:

    •   Les trois mécanismes de base du web sont :
    URL : le mécanisme de localisation qui permet d'accéder à un ensemble de ressources en faisant appel au protocole DNS. HTTP : un protocole de niveau applicatif qui suit le paradigme client-serveur et est sans état, ce qui signifie qu'il ne se souvient pas des transactions précédentes.
    HTML : un langage de description de pages web qui décrit chaque élément de la page, tels que le titre, les morceaux de texte et les images, et qui permet la programmation déclarative et la création de liens hypertextes entre les pages.

  • Page web et site web

    •   Une page web est un document multimédia qui peut contenir des images, des textes, des liens interactifs et des possibilités d'interaction. Elle est décrite élément par élément à l'aide d'un langage de description tel que HTML, XHTML ou HTML5, et stockée dans un fichier HTML. Une page web est interprétée et affichée par un navigateur et est localisée sur Internet à l'aide d'une adresse URL. Elle permet d'accéder à d'autres pages en suivant des liens. Un site web est un regroupement de pages web autour d'une thématique commune, reliées entre elles par des liens hypertextes, émanant d'une même entité telle qu'une organisation, une entreprise ou un particulier. Les sites web sont accessibles à partir d'une page d'accueil commune et d'une URL de base commune, et sont généralement localisés sur un même serveur. Ils sont structurés sous forme d'arborescence de dossiers et de fichiers, ce qui permet de définir des chemins relatifs.

  • Création d'une page web HTML

    •   Un fichier HTML est un fichier texte structuré par ces balises et interprété dans un navigateur. Il peut être créé à l'aide d'un éditeur de pages web comme Dreamweaver ou tout simplement avec un éditeur de texte tel que Bloc-notes ou WordPad.
      Pour créer une page web en HTML, il faut utiliser des balises HTML pour structurer le document. Au minimum, le fichier doit comporter les balises < html >, < head >, < title > et < body >. Pour créer des titres et des paragraphes, on utilise les balises < h1 > à < h6 > et < p >.

  • Démarrer un dépôt Git:

    • 

    Un “dépôt” correspond à la copie et à l’importation de l’ensemble des fichiers d’un
projet dans Git. Il existe deux façons de créer un dépôt Git :
• On peut importer un répertoire déjà existant dans Git ;
• On peut cloner un dépôt Git déjà existant.

    


  • Les états des fichiers

    • 
	•Git suit les modifications des fichiers d'un projet en les classant en deux états : "suivi" et "non suivi".
    	Lors du démarrage d'un dépôt Git ou lors de l'importation d'un répertoire existant, les fichiers sont 
		initialement non suivis et doivent être indexés et validés.
    •Les fichiers suivis peuvent avoir trois états : "modifié", "indexé" et "validé".
    	Un fichier est considéré comme "modifié" tant qu'il n'a pas été indexé, et un fichier nouvellement 
		ajouté est considéré comme "non suivi".
    •L'indexation d'un fichier indique à Git qu'il doit faire partie du prochain instantané.
    •Lorsqu'un instantané est pris, les fichiers indexés et non modifiés sont validés, et le cycle peut 
		recommencer.


    


  • Les zones de travail

    • 

    Le répertoire de travail est où les fichiers du projet sont stockés et
modifiés. La zone d'index est un fichier qui stocke les informations sur
les fichiers qui feront partie du prochain instantané ou commit. 
Le répertoire Git est l'endroit où Git stocke les métadonnées et la base de
données des objets du projet. Le processus de travail consiste à modifier
les fichiers dans le répertoire de travail, les indexer dans la zone
d'index, puis les valider dans le répertoire Git. Ces zones de travail
permettent à Git de suivre les modifications sur les fichiers d'un projet.

    
	
  • Gestion de version en utilisant Git

    • 
    
  • 1-Créer un dépôt Git à partir d'un répertoire existant :
    • 
    
    
      -Utiliser la commande "git init" pour initialiser un dépôt Git.
      -Utiliser "git status" pour connaître l'état des fichiers.
      -Indexer les fichiers avec "git add" pour qu'ils puissent être validés.
      -Valider les fichiers avec "git commit", en ajoutant un message pour
      documenter les modifications.
    
    
	
  • 2.Cloner un dépôt Git :

    • 
    
    
      -Utiliser la commande "git clone" en spécifiant l'URL du dépôt existant.
      -Ajouter des fichiers au dépôt cloné avec "git add" et "git commit".
      -Pousser les modifications vers le dépôt cloné avec "git push".
    
    
	
  • 3-Ajouter ou modifier des fichiers dans un projet et actualiser notre
		dépôt Git :

    • 
    
    
      -Utiliser "git add" et "git commit" pour enregistrer les changements dans
      le dépôt Git.
      -Utiliser "git commit -a" pour mettre en un coup les fichiers modifiés et
      déjà sous suivi dans la zone d'index, et les valider.
      -Répéter ces étapes pour actualiser le dépôt Git avec de nouveaux
      changements.
    
    
	
  • 4-Consulter l'historique des modifications Git et annuler des
		modifications apportées à un fichier 

    • 
    
          -Utiliser la commande "git log" pour consulter l'historique des
      modifications Git.
      -Utiliser "git checkout -- nom-du-fichier" ou "git restore" pour annuler
      les modifications apportées à un fichier et revenir à son état précédent.
    
    
	
  • branche

    • 

    
    
      Une branche dans Git est un pointeur vers un commit spécifique.
      Contrairement aux autres systèmes de contrôle de version, Git ne crée pas
      une copie physique de l'ensemble du répertoire de travail lorsqu'une
      nouvelle branche est créée, ce qui facilite la création et la fusion de
      branches. La branche par défaut dans Git s'appelle "master" et est
      simplement une branche comme les autres. Lorsqu'on crée une nouvelle
      branche, on crée simplement un nouveau pointeur vers un commit plutôt que
      de recopier l'intégralité du répertoire de travail.
